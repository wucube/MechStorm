# ADR 格式

架构决策记录（ADR）记录了重要的架构决策及其背景和后果。

## 文件命名

`NNNN-short-title.md`，其中 `NNNN` 是零填充序号。例如 `0003-use-event-sourcing.md`。

## 格式

```markdown
# N. 简短标题

## 状态

提议 | 已接受 | 已弃用 | 已取代（由 ADR-NNN）

## 背景

描述促使决策的技术或业务力量。
什么问题需要解决？有哪些约束？

## 决策

做出决策并解释理由。
包括被考虑的替代方案以及为何被拒绝。

## 后果

决策带来的结果——包括正面和负面的。
对系统、团队或运维的影响。
```

## 原则

- **ADR 记录决策，而非描述系统。** "我们使用事件溯源"是一个决策。"系统使用事件溯源"是描述。决策格式迫使你解释*为什么*。
- **不可变性。** 一旦 ADR 被接受，就不应修改——只能被后续 ADR 弃用或取代。
- **简短。** 每个 ADR 应该能在 1-2 分钟内读完。如果太长，抽象级别就错了。
- **链接到其他 ADR。** 决策很少孤立存在。引用相关的 ADR 以建立决策谱系。
